Generative text for replicating document voice¶

Problem description¶

The main idea of this project is to replicate the certain style and word choices of a particular document or set of documents. This idea has already been applied to replicating books as well as attempting to replicate poetry. There are many different approaches such as neural networks [1], reinforcement learning [3], and another approach included using variational autoencoders with holistic attribute dis-criminators [5]. Overall, there must be some model that takes in data and uses that data in order to generate sentences. The main approach this project will use is a modified markov model.

Description of other methods¶

One probable method for text generation is through using a neural network. This method uses the same basic principles of a neural network but needs to convert the text into a numerical format in order for the network to understand the data. This conversion can be done through converting the entire word into a numerical format or by converting each character in the word to a number [1]. From the article describing this method, the numeric words were then fed into a long term short memory recurrent neural network. After attempting this model to predict the rest of sample lyrics, there was a fairly low accuracy rate which suggests this may not be the best approach for text generation.

Another probable model for generative text is through using a reinforcement learning (RL) approach. This uses the basic RL updates and algorithm in order to develop the best policy for selecting the next word in a sentence [3]. The updates consist of evaluating how well a sentence was formed at the end of an episode. This calculated reward is then used to manipulate the existing sentence which then converges on a policy. It is possible though that at the end of this algorithm, the policy converges to a single sentence that is generated every time [3]. This approach then makes it difficult to continually generate sentences that all sound relatively different.

A different model for text generation is targeting the idea of controlled sentence construction through the use of variational auto-encoders (VAEs) and holistic attribute discriminators [5]. This approach is essentially using two neural networks for encoding and decoding the sentences which end up learing distributions. The encoder takes in the sentences and outputs a distribution for that sentence. The decoder then tries to do generative work to try and reconstruct sentence x [2]. The attribute discriminators are then used to restrict what type of output the generator is giving back. Overall, this approach was successful in generating controlled sentences but is quite costly to train the networks.

A final probable approach to sentence generation is by using a markov model. This method takes in a sequence of words and builds a basic markov model based on the words that immediately came after a word. Therefore there will be a set S that contains a distribution of words that come after word v $\in$ S. This can then be randomly sampled to develop a sentence based on source material [4].

Modified Markov Model Approach for this Project¶

This technique processes every word in a source text and builds a markov model based on where words showed up in sentences. By using this markov approach, this uses the assumption that each word is only dependent on its predecessor. Therefore, this is assuming that the markov assumption will be valid enough to generate successful sentences. The assumption should be more successful than a random approach. This is because the model will be selecting the next word in the sentence based on words that existed in the training documents. So, when sentences are generated, it will be sampling from a distribution of words that are the most likely to come next in order to create more grammatically correct sentences.

In order to improve the accuracy of this approach, the markov approach has been enhanced to include checking if word triples are accurate as well. In the basic markov approach, each word will have a connection to other words that existed after it in the texts. This approach will select two words ($\textbf{W}_1 , \textbf{W}_2$) based on that model and then select a third word ($\textbf{v}$) from the set S that appears in the following sequence:

$$\textbf{W}_1 \rightarrow \textbf{W}_2 \rightarrow \textbf{v}$$

$$ \text{where } \textbf{v} \in \textbf{S} \text{ and } \textbf{S} \text{ is the set of all words that create a triple, }\textbf{W}_1 \rightarrow \textbf{W}_2 \rightarrow \textbf{S} \text{ which existed in the original text} $$

Then a new word triple will be formed where $\textbf{W}_2$ becomes $\textbf{W}_1$ and $\textbf{v}$ becomes $\textbf{W}_2$. A new word will then be selected from the set that generates a triple using the new $\textbf{W}_1 , \textbf{W}_2$.

By adding this step in the generative process, the sentences should gain more meaning and sound more like the source documents. This is because the words will now check if that three word phrase existed in the original documents which should provide higher accuracy than a simple markov model.

This will then be used to build a markov model. The rest of the code for the markov model with word triple checking is created below.

#helper function to go through a file and return a list of every word with ending punctuation separated
def processFile(file):
    #open the file and remove the first two lines which are just meta data
    speech = open(file, "r")
    speechWords = []
    speech = list(speech)[3:]

    #iterate through each line in the text file
    for line in speech:
        words = line.split()
        for word in words:
            tempString = ""
            possiblePunc = ""
            #check if there is an ending punctuation in the word and split on that
            for char in word:
                if char in endingPunctuation:
                    possiblePunc = char
                else:
                    tempString += char
            speechWords.append(tempString)
            #add the punctuation as a separate word for easier processing later
            if possiblePunc != "":
                speechWords.append(possiblePunc)

    return speechWords

#takes in a list of words from a raw text and returns a dictionary of next word probabilities and starting words
def generateMarkov(text):
    #initialize variables
    markovModel = {}
    startingWords = []
    startingWords.append(text[0])
    previousWord = ""

    #iterate through each word and build a markov model
    for word in text:
        #check if we need to add the current word to the starting words list
        if previousWord in endingPunctuation:
            startingWords.append(word)

        #update the markov dictionary with the new word
        if previousWord in markovModel:
            keyList = markovModel[previousWord]
            keyList.append(word)
            markovModel.update({previousWord: keyList})

        else:
            markovModel.update({previousWord: [word]})

        previousWord = word
    
    #convert the lists into counters for faster processing later
    for key in markovModel:
        keyList = markovModel[key]
        markovModel.update({key: Counter(keyList)})
        
    return startingWords, markovModel

#helper function that returns all possible triples in the list to help coherency
def generateTriples(wordList):
    #initialize variables
    wordTriples = []
    length = len(wordList)
    
    #iterate through each word and build a triple
    for i in range(length - 3):
        newTriple = [wordList[i], wordList[i+1], wordList[i+2]]
        wordTriples.append(newTriple)
    
    return wordTriples

#takes in a list of starting words and a markov model to generate a sentence
def generateSentence(startingWords, markov, wordTriples):
    #initialize the first word
    sentence = random.choice(startingWords)
    numWords = 0
    #initialize the word triples and the first part of the sentence
    newWords = markov[sentence]
    previousWord = random.choice(list(newWords.elements())) #randomly draw from the word list
    currWordTriple = [sentence, previousWord]
    sentence = sentence + " " + previousWord

    #iterate through the markov chain until an ending punctuation has been reached
    while(previousWord not in endingPunctuation):
        validNextWords = Counter()
        nextWords = markov[previousWord]
        
        #check if the next word is a possible valid word by checking if the word triple existed in the documents
        for word in nextWords:
            if currWordTriple + [word] in wordTriples:
                validNextWords.update({word: nextWords[word]})

        #choose the next word from the associated list of valid next words
        if len(validNextWords) == 0:
            #randomly choose from all words if the triple broke down
            currWord = random.choice(list(nextWords.elements())) 
        else:
            currWord = random.choice(list(validNextWords.elements()))
            
        #build the next word triple to compare for the next iteration
        currWordTriple = [currWordTriple[1]] + [currWord]
        
        #add the new word to the current sentence and continue
        if currWord in endingPunctuation:
            sentence = sentence + currWord
        else:
            sentence = sentence + " " + currWord
            
        previousWord = currWord

    return sentence

The output sentences from the markov model are mostly coherent and are able to have meaning extracted from them. This shows that the markov model is a better approach than simply randomly selecting the next word for content generation. This is a better model to compare to since there is more structure to the generated content. Since this new baseline model does not need to go through and double check sentence triples, there is also an increase in speed for the content generation. Comparisons to the simple markov baseline

The new baseline model already has far better sentence structure and grammar than the naive baseline model of random sentence generation. However, there are still many grammar mistakes and many of the sentences have little to no substance. The markov model with word triples allows for better sentence generation overall with more meaning able to be extracted from the sentences. Conclusions¶

The Markov Model with word triple checking was an effective choice in generating sentences that are similar to a particular style such as the presidential speeches. Adding in the idea of checking word triples was successful in terms of the content generated since there is a greater chance of the sentences being more grammatically correct if the word triple appeared in the original text. This does however make the generation of the sentences take more time since it needs to go through and double check if the next word added to the sentence will generate an appropriate word triple or not. This project could also be improved by adding in some grammatical system to generate sentences that are more or less guarunteed to be grammatically correct. While the current model generates sentences that have more significant meaning than the random model, there are still sentences that are not grammatically correct. Therefore, by adding in a grammar system, this could greatly benefit the overall generative process.

In terms of how this approach compares to other processes such as neural networks or variational auto-encoders, this model takes significantly less time to build. While a neural network could take 15 minutes to train, this approach takes mere seconds. However, the content generated is more likely to make more sense coming from a NN as opposed to this basic markovian model. Overall, the content generated through this approach is nonetheless entertaining and does indeed sound like sentences that could have appeared in the presidential speeches. Also, the data from the grammatical mistake analysis shows that the markov model with word triple checking is a viable method which generates sentences with a reasonable degree of accuracy.
